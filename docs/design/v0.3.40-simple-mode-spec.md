## Components and behavior

- `ViewModeSwitch`: two accessible buttons with selected state, keyboard focus, and per-Windows-user persistence under `pos-printer-emulator-view-mode`.
- `SimpleWorkspace`: health summary, numbered setup workflow, connection fields, latest receipt, quick actions, and Expert Mode handoff.
- `ConnectionValue`: read-only listener value plus Copy action. Copy feedback must be announced without exposing any additional network or license data.
- `SimpleLatestReceipt`: reuse the real `ReceiptPaper` renderer and the currently selected/latest receipt; selecting a job in Expert Mode remains selected when modes change.
- Quick actions call the existing capture import input, Settings printer wizard, Settings support diagnostics/request section, and Test Receipt API. Existing license checks stay enforced by current controls and service APIs.
- Switching modes must preserve selected receipt, search, listener filter, inspector tab, zoom, collapsed panels, theme, and listener context.

## Responsive rules

- At desktop widths, use a two-column main region and a full-width quick-action rail.
- Below 980 px, stack Latest Receipt below the workflow and wrap header controls without removing the mode switch.
- Below 700 px, stack connection values and quick actions, maintain 44 px minimum primary targets, and allow the workspace to scroll vertically.
- Respect `prefers-reduced-motion`; no new decorative animation is required.

## Acceptance

- All four license tiers can use and persist both modes.
- Simple Mode never changes or bypasses license authorization.
- Every Simple Mode action reaches a working existing workflow.
- Switching modes preserves the active receipt and existing Expert Mode state.
- Light/dark themes, keyboard focus, small-laptop layout, and the no-job/latest-job states render without clipping.
